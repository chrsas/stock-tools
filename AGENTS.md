# 协作规范

1. 标题和列表适度使用，避免过度分层。
2. 减少使用表情符号。
3. 禁止使用“不是……而是……”句式。
4. 避免使用破折号。
5. 搜索资料时尽量减少使用中文网站信源。
6. Git 提交信息使用中文，简洁描述本次改动目的。
7. 开工前通读 `tasks.md` 第 0 节与第 6 节，再读当前要做的阶段一节；改动采集、状态机、数据库或适配器前先读 `docs/architecture.md`（技术栈、数据模型、状态机细节）；已完成阶段的详细规范在 `docs/phases-archive.md`，仅供追溯，日常开发无需阅读。每完成一个阶段，将其详细规范移入归档，`tasks.md` 原位置只留一行交付摘要。
8. 提交前运行 `.\scripts\check_quality.ps1`，确保 lint、格式检查、类型检查和自动化测试全部通过。
9. 提交前检查 `git status` 和差异内容，确保凭据、本地配置、运行数据和原始采集文件未进入提交。
