# 协作规范

1. 标题和列表适度使用，避免过度分层。
2. 减少使用表情符号。
3. 禁止使用“不是……而是……”句式。
4. 避免使用破折号。
5. 搜索资料时尽量减少使用中文网站信源。
6. Git 提交信息使用中文，简洁描述本次改动目的。
7. 开工前通读 `tasks.md` 第 0 节，并阅读当前阶段对应任务；涉及代码时同时遵守 `tasks.md` 第 6 节。
8. 提交前运行 `.\scripts\check_quality.ps1`，确保 lint、格式检查、类型检查和自动化测试全部通过。
9. 提交前检查 `git status` 和差异内容，确保凭据、本地配置、运行数据和原始采集文件未进入提交。
